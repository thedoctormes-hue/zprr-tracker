## Алгоритм дедупликации проектов

### Критерии "ЖИВОЙ" проект:
1. **Git activity** — коммиты за последние 7 дней
2. **Systemd unit файлы** — service/timers в /etc/systemd/system/
3. **Production сервисы** — uvicorn/fastapi на портах, работающие процессы
4. **Актуальные даты файлов** — основные .py файлы < 7 дней
5. **Тесты и кэш** — .pytest_cache/, tests/conftest.py

### Критерии "МЁРТВЫЙ" backup:
1. **Нет git репозитория**
2. **Нет systemd сервисов**
3. **Файлы .backup, .backup2, .backup3**
4. **Дата создания > 30 дней**
5. **Отсутствие .pytest_cache**

### Checklist для скрипта:
- [ ] Найти все папки с одинаковым названием проекта
- [ ] Сравнить даты последних git коммитов
- [ ] Проверить наличие systemd unit файлов
- [ ] Сравнить размер и даты файлов
- [ ] Архивировать мёртвый в /archive/
- [ ] Удалить пустые директории

### Применение:
- Автоматизировать в `self_evolve.sh`
- Интегрировать в CI для предупреждения о дублях
