## 🔥 Критические выводы

### 1. Паттерн fallback Markdown — обязательный минимум
**Проблема:** Telegram иногда отказывается парсить символы в неожиданных местах
**Решение:** Каждая функция с `parse_mode="Markdown"` должна иметь try/except:

```python
try:
    await message.answer(text, parse_mode="Markdown")
except Exception:
    await message.answer(text.replace('**', '').replace('__', ''))
```

### 2. Дублирование команд — системная проблема
**Примеры:**
- `/start` + `/help` — 90% дублирования
- `/optimize` + `/recommend` — 90% дублирования
**Решение:** Объединять в одну функцию через декоратор `@dp.message(Command("cmd1")) @dp.message(Command("cmd2"))`

### 3. MarkdownV2 требует отдельного подхода
**Проблема:** `mail-daemon/main.py` использует MarkdownV2
**Решение:** Экранирование другое: `_ * [ ] ( ) ~ \` > # + - = \| { } . !`
