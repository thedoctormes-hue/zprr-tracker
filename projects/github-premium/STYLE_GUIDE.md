# 📐 Стандарт оформления репозиториев LabDoctorM

Единый гайд по структуре, именованию и автоматизации репозиториев лаборатории ЗавЛаб Безумный Доктор.

---

## 📂 Обязательная структура репозитория
Каждый репозиторий должен содержать:
```
repo-root/
├── README.md          # Основная документация (шаблоны в /templates)
├── .gitignore        # Исключения для Git (сгенерированный или кастомный)
├── .github/
│   ├── workflows/    # GitHub Actions (шаблоны в /workflows)
│   ├── ISSUE_TEMPLATE.md
│   └── PULL_REQUEST_TEMPLATE.md
├── tests/            # Тесты (для кодовых проектов)
└── docs/             # Дополнительная документация (опционально)
```

---

## 📄 Стандарт README.md
Все README должны следовать структуре:
1. **Заголовок с эмодзи** (название проекта + краткий дескриптор)
2. **Бейджи** (обязательные):
   - ![Build Status](https://img.shields.io/github/actions/workflow/status/OWNER/REPO/python-app.yml?label=build)
   - ![Version](https://img.shields.io/github/v/release/OWNER/REPO)
   - ![License](https://img.shields.io/github/license/OWNER/REPO)
   - ![Stars](https://img.shields.io/github/stars/OWNER/REPO?style=social)
3. **TL;DR** (1-2 предложения о назначении проекта)
4. **🎯 Цели** (список ключевых задач проекта)
5. **⚡ Features** (список реализованного функционала)
6. **🚀 Quick Start** (инструкции по запуску локально)
7. **📚 API** (для бэкенд-проектов, опционально)
8. **🤝 Contributing** (правила участия в разработке)
9. **📞 Контакты** (канал поддержки, чат разработки)

---

## 🌳 Стандарты именования веток
- `feature/*` — новый функционал
- `fix/*` — исправление багов
- `chore/*` — обслуживание (обновление зависимостей, конфигов)
- `docs/*` — изменения документации

---

## 📝 Стандарты коммитов (Conventional Commits)
Обязательный формат:
```
<type>(<scope>): <description>

[optional body]

[optional footer]
```
Где `type`:
- `feat:` — новый функционал
- `fix:` — исправление ошибки
- `docs:` — изменения документации
- `chore:` — обслуживание инфраструктуры
- `test:` — добавление/исправление тестов

---

## ⚙️ Требования к CI/CD
### Для Python-проектов (боты, FastAPI):
- Запуск линтера (ruff)
- Запуск тестов (pytest)
- Проверка валидности requirements.txt

### Для React/Node-проектов:
- Установка зависимостей
- Сборка проекта (`npm run build`)
- Запуск тестов (`npm test`)

### Для лендингов:
- Сборка статического контента
- Автодеплой на GitHub Pages

---

## 🎨 Правила использования бейджей
- Используйте [Shields.io](https://shields.io) для генерации бейджей
- Обязательные бейджи: статус сборки, версия релиза, лицензия, количество звезд
- Не более 5 бейджей в шапке README

## 📋 Шаблоны issue и PR
### Issue должен содержать:
- Описание проблемы/задачи
- Шаги воспроизведения (для багов)
- Ожидаемый результат
- Скриншоты/логи (опционально)

### PR должен содержать:
- Ссылку на связанный issue
- Описание внесенных изменений
- Скриншоты (для фронтенда)
- Чекбокс: "Протестировано локально"
