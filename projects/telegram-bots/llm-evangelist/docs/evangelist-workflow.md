---

## 🎯 Кто такой LLM Evangelist?

**LLM Evangelist** (LLM-евангелист) — это специалист, который:
- 📊 Мониторит рынок LLM моделей
- 🎯 Помогает выбирать оптимальные модели под задачи
- 👨‍👩‍👧‍👦 Анализирует семьи моделей и их особенности
- 🏆 Формирует субъективные рейтинги качества

---

## 🔄 Daily Discovery Workflow

### Утренний ритуал (9:00)

```bash
# 1. Запуск ежедневного скрининга
python main.py

# 2. Проверка канала @LLMevangelist
# Бот сам опубликует отчёт в 9:00
```

### Что анализировать ежедневно?

| Показатель | Где смотреть | Как интерпретировать |
|------------|--------------|---------------------|
| Новые модели | OpenRouter /models | Новые версии = улучшения |
| Цены | pricing endpoint | Падение цен = конкуренция |
| FREE модели | filter `:free` | Рост количества = доступность |
| Скорость | timing запросов | Медленнее = нагрузка |

### Инструменты мониторинга

**Telegram бот:**
- `/models` — текущий список моделей
- `/top` — топ по цена/качество
- `/scan` — что используют в лаборатории

**Внешние источники:**
- OpenRouter Blog — анонсы новых моделей
- Hugging Face — исследования и сравнения
- Twitter/X — отзывы разработчиков

## 🎯 Task Profiling — Как выбрать модель

### Алгоритм выбора модели

```
┌─────────────────┐
│   Задача определена  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Выбрать тип задачи │
│  (чат, код, анализ) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Определить бюджет │
│   (free / paid)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Выбрать семью моделей │
│   (qwen, deepseek, и т.д.) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Тестировать топ-3 │
│   через /compare    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Выбрать победителя │
└─────────────────┘
```

### Критерии выбора по задачам

#### 💬 Чат-боты
- **Важно:** скорость ответа, контекстное понимание
- **Тест:** диалог с 10 вопросами подряд
- **Лучшие семьи:** Qwen, Mistral, Claude

#### 💻 Программирование
- **Важно:** точность синтаксиса, знание библиотек
- **Тест:** задача на реальном коде
- **Лучшие семьи:** Qwen2.5-Coder, DeepSeek, Codestral

#### 📊 Анализ данных
- **Важно:** работа с таблицами, статистика
- **Тест:** CSV данные + вопросы
- **Лучшие семьи:** Gemini, Claude, Llama-4

#### 📝 Креативный контент
- **Важно:** фантазия, стиль, гибкость
- **Тест:** "Напиши историю про..."
- **Лучшие семьи:** Mistral, Gemini, GPT-4

## 👨‍👩‍👧‍👦 Family Analysis — Разбор семейств

### Как анализировать семью моделей?

**Шаг 1: Определить фундамент**
- Кто разрабатывает? (компания/исследовательская группа)
- Какие задачи решает базовая модель?

**Шаг 2: Сравнить версии**
- Large vs Medium vs Small
- Как меняется соотношение цена/качество?

**Шаг 3: Проанализировать варианты**
- Бесплатные (:free) — для тестирования
- Платные — для продакшена
- Специализированные (coder, vision и т.д.)

### Шаблон анализа семьи

```markdown
## Семья: Qwen

**Разработчик:** Alibaba

**Базовые модели:**
- qwen-3-30b-a3b — универсальная
- qwen2.5-coder-32b — для кода
- qwen2.5-vl-72b — vision

**Сильные стороны:**
- Многоязычие (CN, EN, RU)
- Открытый код
- Хорош для кода

**Слабые стороны:**
- Медленнее американских конкурентов
- Меньше документации на русском

**Рекомендации:**
- Free: qwen-3-30b-a3b:free для старта
- Paid: qwen2.5-coder-32b для production кода
```

## 🏆 Personal Rankings — Субъективные оценки

### Как ставить оценки?

**Оценка 5 звёзд:**
- Отлично подходит под задачу
- Хорошее цена/качество
- Стабильная работа

**Оценка 4 звезды:**
- Подходит, но есть нюансы
- Цена выше средней

**Оценка 3 звезды:**
- Базовые функции работают
- Есть заметные недостатки

**Оценка 2 звезды:**
- Работает, но плохо
- Частые ошибки

**Оценка 1 звезда:**
- Не рекомендуется к использованию

### Формат записи ранжирования

```
### Gemini 2.0 Flash

⭐⭐⭐⭐☆ 4/5

**Плюсы:**
- Очень быстрая
- Хороший creative mode
- Дешево ($0.1/1M)

**Минусы:**
- Иногда глючит на длинных контекстах
- Меньше tool use чем у Claude

**Использование:** для интерактивных чатов
```

## 🆘 Быстрая справка

**Не могу выбрать модель?**
1. Определить тип задачи
2. Запустить `/compare free [запрос]`
3. Выбрать лучший ответ

**Нужна новая модель?**
1. Проверить `/models`
2. Найти среди FREE
3. Протестировать через `/compare`

**Оптимизировать затраты?**
1. `/analyze` — рекомендации
2. Перейти на FREE модели
3. Отслеживать через `/stats`
