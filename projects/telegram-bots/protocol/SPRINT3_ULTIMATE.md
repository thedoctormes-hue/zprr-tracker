## 5. 📄 Технический контекст проекта Protocol

### Стек технологий
- **Backend:** FastAPI 0.115.0 + Uvicorn
- **Bot:** aiogram 3.x
- **Database:** SQLite 3 + FTS5 (Full-Text Search)
- **Auth:** JWT (HS256, 30 дней), Rate Limiting (slowapi)
- **AI:** OpenRouter (Gemini 2.5 Flash / Llama 3.3 70B)
- **Audio:** ffmpeg + Gemini Flash Lite (транскрибация)

### Архитектура файлов
- `main.py` — Точка входа FastAPI, CORS, Rate Limiter.
- `bot/main.py` — Telegram-бот (@Protocolstandbot), **все кнопки активны ✅**.
- `app/database.py` — Схема БД, FTS5, функции `get_people_edges`.
- `app/classifier.py` — Классификация фрагментов и транскрибация.
- `app/night_analyst.py` — Ночной анализ паттернов (00:00 UTC).
- `app/routers/edges.py` — **NEW:** CRUD для `memory_edges`.
- `app/routers/settings.py` — **NEW:** API для `exit_settings`.

### Статус Спринтов 1-2
- ✅ Спринт 1: Безопасность (JWT, Rate Limit, CORS, Backups).
- ✅ Спринт 2: Активация фич (Edges API, Settings API, Кнопки бота, Тесты).

## 7. ⚙️ Инструкции по запуску и деплою

### Быстрый старт
```bash
# 1. Проверка сервисов
systemctl status protocol.service protocol-bot.service

# 2. Логи бота
journalctl -u protocol-bot.service -n 50 -f

# 3. Тесты (ОБЯЗАТЕЛЬНО перед коммитом)
cd /root/protocol && /root/LabDoctorM/venv/bin/pytest tests/ -v

# 4. Бэкапы
ls -la /root/protocol/backup/
```

### Деплой (через deploy-bot или вручную)
1. Установка зависимостей: `/root/LabDoctorM/venv/bin/pip install -r requirements.txt`
2. Перезапуск бота: `systemctl restart protocol-bot`
3. Перезапуск API: `systemctl restart protocol.service`

## 8. 🤝 Как работать с сотрудниками и скиллами

### Вызов сотрудника
Для вызова специалиста используйте команду вызова сотрудника (если настроено) или обращайтесь через контекст.
Пример: *"Вызови сотрудника hr для найма специалиста по безопасности"*.

### Использование скиллов
Скиллы вызываются через `/skill <name>` или по триггер-фразам.
Пример: *"Сделай документацию для проекта Protocol"* (вызовет `create-readme`).

### Важные напоминания
- **JWT работает:** Используйте `Authorization: Bearer <token>`, забудьте про `X-TG-ID`.
- **Тесты:** Всегда запускайте `pytest` перед деплоем.
- **Контекст:** Для восстановления контекста используйте `context-session-specialist` или читайте `TRANSITION.md`.
